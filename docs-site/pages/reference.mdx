# Reference

## Tool return format

All tools return a consistent MCP content structure:

```typescript
{
  content: [{ type: "text", text: JSON.stringify(result, null, 2) }]
}
```

On error:

```typescript
{
  content: [{ type: "text", text: "Error: <message>" }],
  isError: true
}
```

Claude parses the JSON automatically and reasons about the result.

## Encoding utilities

### Lovelace to ADA

```typescript
import { lovelaceToAda } from "./config.js";
lovelaceToAda(2000000n) // → "2.000000"
```

### Decode CBOR datum

```
decode_cbor_datum({ datum: "d87980" })
// → { "constructor": 0, "fields": [] }
```

### Encode Plutus data

```
encode_plutus_data({ data: { constr_index: 0, fields: [] } })
// → "d87980"
```

## Error codes

| Error | Cause |
|-------|-------|
| `404 Not Found` | Address/tx/asset does not exist on the selected network |
| `402 Usage limit reached` | Blockfrost daily limit exceeded |
| `429 Too Many Requests` | Rate limited — Koios client retries automatically |
| `CBOR decode error` | Malformed or non-Plutus CBOR datum |
| `aiken not found` | Aiken CLI not installed — needed for validate_aiken_snippet |
